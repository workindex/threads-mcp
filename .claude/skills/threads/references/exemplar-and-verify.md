# 예문 선별 + 검증 — Layer B 질감 & 게이트

> 출처: `Hiro-Inagawa/write-like-me`(12★) 05·07 적응.
> 핵심 근거(Patel et al., 2023): 정량 프로필만으론 부족. **실제 예문 3~5개**가 질감을 옮긴다. 추출·요약하면 그 효과가 사라진다(= 맨 처음 실패 원인).

## A. 예문 선별 (초안 작성 전)

`get_draft_context` Top20에서 **오늘 소재·포맷과 가장 가까운 3~5개**를 고른다. **점수순이 아니라 유사도순.**

선별 기준:
1. 오늘과 같은 종류의 글 (후기면 후기, 공지면 공지, 깨달음이면 깨달음)
2. 같은 레지스터 (오늘이 습니다체면 습니다체 예문)
3. 사장님 특징이 가장 진한 글 (아웃라이어 말고 전형)
4. 서로 다른 무브를 covering (훅·전개·종결이 다 같은 것 X)

**hold-out**: 가장 가까운 글 1~2개는 예문으로 쓰지 말고 **검증용으로 빼둔다.**

선별한 예문은 그대로 모방한다. 어미·줄바꿈·종결을 베낀다. "참고"가 아니라 "이식".

## B. held-out 검증 (초안 작성 후)

빼둔 hold-out 글과 초안을 대조. 각 항목 pass/fail:

| 체크 | 통과 조건 |
|------|-----------|
| 어미 일관성 | 한 레지스터로 통일 (섞임 없음) |
| 첫 줄 길이 | hold-out 훅과 비슷한 짧기 |
| 문장 길이대 | 평균이 코퍼스 범위(짧음) 안 |
| 말줄임표/물음표 | 감정·질문 맥락이면 존재 |
| 대구·이항대비 | 0개 (anti-slop 1절 위반 0) |
| 길이 모드 | 초단문 또는 150~400자, 중간 어정쩡 X |
| 이모지 | 0~1개 |

**2개 이상 fail → 1회 수정(최대 2회).** 같은 체크에서 반복 실패하면 그 줄을 하드 제약으로 다시 쓴다.

## 검증이 보장하지 않는 것

- 글이 "좋은가" — 사장님 판단.
- 사람과 구분 불가한가 — 문체측정의 한계.
